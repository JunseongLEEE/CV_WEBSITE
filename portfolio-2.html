FINDER

Description

  • Real-time information service for lost items on the subway and matching service with the finder.

  • GITHUB

  • 2023 06 26

Implementation

  • Finance API: Users looking for lost items pay a certain fee through the Kakao/Toss API in the app, and users who find the lost items receive a reward through the API.
  • Map API: The mashup server receives address (location) data through the Road Name Address API and provides users with a location to retrieve their lost items via the map API.
  • Lost Item API: Before users register a lost item on the web app, the API checks if their item matches any pre-registered lost items.

WHY FINDER?

  • There are no alternatives to reporting lost items other than to the police, and it is rare for found items to be returned to their owners.
  • Existing lost item data is hard to access, and differing locations of loss and discovery make finding items challenging.
  • We aim to develop a comprehensive lost item retrieval service to address the issues of insufficient rewards and lack of time.
